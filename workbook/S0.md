# S0 — The problem: getting the static line into Excel

The static buy-and-hold line ("Step 0" of the storyboard — the thesis opener) ships as
data in the GitHub release **[data-v2](https://github.com/norandom/Global_Macro_AI_Factors/releases/tag/data-v2)**.
Three ways to load it, easiest first. No repository checkout needed for any of them.

## 1. CSV via URL (any Excel, zero setup)

**Data → Get Data → From Web**, paste one of:

```
https://github.com/norandom/Global_Macro_AI_Factors/releases/download/data-v2/static_bh_equity_2016_2026.csv
https://github.com/norandom/Global_Macro_AI_Factors/releases/download/data-v2/static_bh_equity_2014_2024.csv
https://github.com/norandom/Global_Macro_AI_Factors/releases/download/data-v2/static_bh_targets_2014_2024.csv
```

- The equity CSVs carry `Date, value, daily_return, drawdown` — chart-ready
  (equity curve + drawdown panel; the drawdown column makes the COVID-2020 and
  inflation-2022 episodes directly visible).
- `static_bh_equity_2016_2026` is notebook 04's original 10-year window;
  `static_bh_equity_2014_2024` aligns date-for-date with the dynamic walk-forward
  lines (`factor_equity_v1` etc.).
- The targets CSV is the **drifting** weights — buy-and-hold holds shares, not
  weights (a stacked-area chart of it shows the drift).
- **Data → Refresh All** re-pulls from the release. The URL pins `data-v2`, so the
  numbers cannot shift underneath a written thesis.

## 2. Parquet via Power Query (Microsoft 365)

For the canonical parquet assets: **Data → Get Data → From Other Sources → Blank
Query**, then in the formula bar:

```
= Parquet.Document(Web.Contents("https://github.com/norandom/Global_Macro_AI_Factors/releases/download/data-v2/static_bh_equity_2014_2024.parquet"))
```

Same pattern for any release asset. The headline/caption numbers live in
`static_bh_stats.json` (load with **From Web** → JSON): per-window metrics, the
**crisis episodes** (COVID 2020: max drawdown −20.7% vs SPY −33.7%; inflation 2022:
−20.2% vs −24.5%), the line's own **Sharpe Stability Ratio (0.147 ≪ 1.96 —
luck-compatible)**, and the in-sample selection caveat verbatim.

## 3. The PyXLL workbook (S0 sheet)

The generated workbook (see `README.md`) includes an **S0** sheet: set the release
tag in `Index!B1` to `data-v2` and the sheet loads the static equity/targets/stats
through the same `FW_*` functions as the other steps, with the published metrics
re-derived and checked in front of you (`FW_CHECKS`). Note S0 requires `data-v2`
or later — on `data-v1` the S0 cells show a per-asset `#ERROR` (the assets did not
exist yet), by design.

## Reading the numbers honestly

The two claims must be kept separate (they carry different evidential weight):

1. **The crisis-drawdown episodes are real, event-level observables** — the timing,
   depth, and co-movement with the 2020 and 2022 crises carry no selection artifact.
2. **The performance level is hindsight-flattered** — the four ETFs were selected by
   SSR computed over the same window being simulated (notebook 04's own caveat), and
   the line's own SSR of 0.147 means the gorgeous decade is statistically
   indistinguishable from luck.

Separating those two — visible resilience vs. certifiable skill — is what the rest
of the storyboard (S1–S5) measures.
